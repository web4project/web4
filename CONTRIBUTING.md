# Contributing

Thanks for helping improve **Web4Project**.

## Ground rules

- **Privacy first**: Do not introduce analytics/telemetry or any behavior that sends vault data off-device.
- **Security matters**: Crypto changes must be justified and reviewed carefully.
- **Keep it offline-first**: Features should work without a backend.

## Development setup

```bash
cd web4
npm install
npm run dev
```

## Quality checks

```bash
cd web4
npm run lint
npm run typecheck
npm run build
```

## What to include in a PR

- Clear description of the change and the motivation
- Screenshots for UI changes
- Notes on any security/privacy impact (even if "none")

## Reporting security issues

Please **do not** open public issues for vulnerabilities. See `SECURITY.md`.
